## Overview

This is a comprehensive API documentation for frontend developers. It includes all endpoints with complete JSON request/response examples.

**Base URL:** `https://mifumosms.mifumolabs.com/api/`
**API Version:** 1.0
**Authentication:** JWT Bearer Token

## Quick Start

All API endpoints require authentication using JWT Bearer tokens. Include the access token in the Authorization header:

```
Authorization: Bearer {access_token}
```

The access token is obtained from the [Login endpoint](/api-reference/authentication/login).

## API Structure

The API is organized into the following sections:

- [Authentication](/api-reference/authentication/register) - User registration, login, and account management
- [User Management](/api-reference/user-management/profile) - Profile and settings management
- [SMS Operations](/api-reference/sms-operations/send-sms) - Send SMS, check balance, and delivery status
- [Contact Management](/api-reference/contact-management/list-contacts) - Manage contacts and lists
- [Campaign Management](/api-reference/campaign-management/list-campaigns) - Create and manage SMS campaigns
- [Template Management](/api-reference/template-management/list-templates) - SMS message templates
- [Billing & Payments](/api-reference/billing-payments/sms-packages) - Purchase SMS packages and manage payments
- [Tenant Management](/api-reference/tenant-management/list-tenants) - Multi-tenant organization management
- [Notifications](/api-reference/notifications/list-notifications) - System notifications
- [Sender ID Management](/api-reference/sender-id-management/list-sender-ids) - Manage sender IDs
- [Dashboard & Analytics](/api-reference/dashboard-analytics/overview) - Analytics and reporting

## Authentication Headers

All authenticated endpoints require the following header:

```
Authorization: Bearer {access_token}
```

The access token is obtained from the login endpoint and should be included in all subsequent requests.

## Rate Limiting

API requests are rate-limited to prevent abuse:

- **Authenticated requests**: 1000 requests per hour
- **Unauthenticated requests**: 100 requests per hour

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1638000000
```

## Pagination

List endpoints support pagination with the following query parameters:

- `page` (integer): Page number (default: 1)
- `page_size` (integer): Items per page (default: 20, max: 100)

Pagination response format:

```json
{
  "count": 150,
  "next": "https://api.example.com/endpoint/?page=2",
  "previous": null,
  "results": [...]
}
```

## Date Formats

All dates are returned in ISO 8601 format:

```
2025-11-27T10:30:00Z
```

When sending dates in requests, use the same format or:
- Date only: `2025-11-27`
- Date and time: `2025-11-27T10:30:00Z`

## Phone Number Format

Phone numbers should be in E.164 format:

- ✅ Correct: `+255614853618`
- ❌ Incorrect: `0614853618`, `255614853618`

The API will attempt to normalize phone numbers, but it's recommended to send them in E.164 format.
